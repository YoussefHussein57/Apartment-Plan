# خطة تجهيز الشقة — تطبيق ويب

تطبيق React (Vite + TypeScript) متصل بقاعدة بيانات Supabase، بديل لملف الإكسل يمكن الوصول إليه من أي مكان.

## الإعداد

1. أنشئ مشروعًا مجانيًا على [supabase.com](https://supabase.com).
2. من **SQL Editor** داخل المشروع، شغّل محتوى الملف `supabase/schema.sql` مرة واحدة (ينشئ الجداول ويضيف بيانات الخطة الحالية).
3. من **Project Settings → API**، انسخ:
   - `Project URL`
   - `anon public key`
4. انسخ `.env.example` إلى `.env` واملأ القيمتين:
   ```
   VITE_SUPABASE_URL=...
   VITE_SUPABASE_ANON_KEY=...
   ```
5. ثبّت الحزم وشغّل السيرفر المحلي:
   ```
   npm install
   npm run dev
   ```

## النشر (للوصول من أي مكان)

انشر مجلد المشروع على [Vercel](https://vercel.com) أو [Netlify](https://netlify.com) (كلاهما مجاني):

- اربط المستودع أو ارفع المجلد مباشرة.
- أضف نفس متغيرين البيئة (`VITE_SUPABASE_URL`, `VITE_SUPABASE_ANON_KEY`) في إعدادات المشروع على المنصة.
- الأمر: `npm run build`, مجلد الإخراج: `dist`.

## ملاحظة أمان

لا يوجد تسجيل دخول في هذه النسخة (حسب الاختيار) — أي شخص يملك رابط الموقع المنشور يمكنه قراءة وتعديل البيانات. لا تُشارك الرابط أو ملف `.env` علنًا. يمكن إضافة تسجيل دخول عبر Supabase Auth لاحقًا إذا احتجت حماية أقوى.

## البنية

- `src/pages/Dashboard.tsx` — الملخص العام + توزيع الأقسام + رسم بياني.
- `src/pages/SectionPage.tsx` — صفحة عامة تُستخدم للتشطيب/العفش/الأجهزة (تعديل الميزانية، الحالة، الأولوية، الملاحظات مباشرة في الجدول).
- `src/pages/PaymentsPage.tsx` — تسجيل كل دفعة وربطها ببند.
- `src/pages/SettingsPage.tsx` — القيم المركزية (المرتب، الجمعيات، إلخ).
- `supabase/schema.sql` — الجداول + View لحساب المدفوع/المتبقي/نسبة الإنجاز تلقائيًا + بيانات الخطة الأصلية.

## نطاق النسخة الحالية

هذه نسخة أولى (MVP) تغطي: الأقسام الثلاثة، المدفوعات، الإعدادات، ولوحة تحكم بالإجماليات والتوزيع حسب القسم — وهو ما طُلب مباشرة. جدول "الخطة الشهرية" وتتبع الجمعيات التفصيلي (كما في الإكسل) غير مُنفّذين بعد، ويمكن إضافتهما لاحقًا كخطوة تالية إذا احتجتهما.
